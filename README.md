# Трекер привычек — синк через Supabase, хостинг на GitHub Pages

Личный трекер: один аккаунт, данные синкаются между телефоном, ноутом и компом,
потому что все устройства читают/пишут одну базу в Supabase.

---

## Что где живёт

- **Фронт (это приложение)** — статика, лежит на GitHub Pages.
- **Данные + вход** — Supabase (Postgres + Auth).
- Синк работает так: база одна, устройств много. Залогинился тем же аккаунтом — видишь те же данные.

---

## Шаг 1. Supabase

1. Зайди на supabase.com, создай проект. Регион выбери поближе — например EU (Frankfurt).
   Пароль базы, который попросят, — это для прямого доступа к Postgres; в приложении он не нужен, просто сохрани.
2. Когда проект создастся: **SQL Editor → New query**, вставь содержимое `schema.sql`, нажми **Run**.
3. **Project Settings → API** (или кнопка «Connect»): скопируй
   - `Project URL`
   - `Publishable key` (вид `sb_publishable_...`)
   и вставь оба в `src/supabaseClient.js`.
4. **Authentication → Sign In / Providers → Email**: выключи «Confirm email».
   Тогда регистрация и вход срабатывают сразу, без письма-подтверждения (для личного проекта это ок).

## Шаг 2. Запуск локально

```bash
npm install
npm run dev
```

Откроется на http://localhost:5173 — зарегистрируйся своим email и паролем.
Добавь пару привычек, отметь — проверь, что всё сохраняется.

5. **Закрой регистрацию для чужих.** После того как создал СВОЙ аккаунт, вернись в
   Supabase → Authentication и отключи возможность новой регистрации
   (Allow new users to sign up / Disable signup). Твои данные и так закрыты RLS —
   это просто чтобы посторонние не заводили аккаунты в твоём публичном проекте.

## Шаг 3. Деплой на GitHub Pages

> ⚠️ На бесплатном плане GitHub Pages работает **только из публичного репозитория**.
> Код будет виден всем — это нормально: в нём только publishable-ключ (он публичный
> по дизайну), а доступ к данным защищён политиками RLS. Secret-ключ в коде НЕ хранится.

1. Создай **публичный** репозиторий на GitHub.
2. Залей проект:
   ```bash
   git init
   git add .
   git commit -m "habit tracker"
   git branch -M main
   git remote add origin https://github.com/ТВОЙ-ЛОГИН/РЕПО.git
   git push -u origin main
   ```
3. Собери и выложи:
   ```bash
   npm run deploy
   ```
   (соберёт `dist` и зальёт в ветку `gh-pages`)
4. На GitHub: **Settings → Pages → Source: Deploy from a branch → ветка `gh-pages` → `/ (root)` → Save.**
5. Через минуту сайт живёт на `https://ТВОЙ-ЛОГИН.github.io/РЕПО/`.
   Открой на телефоне → «Добавить на экран Домой» — будет как приложение.

Каждый раз, когда что-то поменяешь в коде: `npm run deploy` снова.
